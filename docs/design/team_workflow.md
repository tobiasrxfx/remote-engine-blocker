# Team Workflow

## Branches
- main: stable branch
- dev: integration branch
- feature/<name>
- fix/<name>
- docs/<name>
- test/<name>

## Commit convention
Use Conventional Commits:
- feat:
- fix:
- docs:
- test:
- refactor:
- build:
- style:
- chore:

## Pull requests
- PRs target `dev`
- at least one peer review
- build must pass before merge
